<h2><a class="anchor" id="overview"></a>Overview</h2>

I built this project to have a lightweight, no-nonsense way to keep track of daily expenses. It’s built entirely with HTML, CSS, and JavaScript. It features a modern dark mode interface (with some sleek purple accents) and uses the browser's local storage, so you don't lose your financial list if you accidentally refresh the page or close your browser. It automatically calculates and updates your total spending as you add or remove items.

<h2><a class="anchor" id="problem-statement"></a>Problem Statement</h2>

Most finance and budgeting apps are way too complicated. You have to create an account, link bank details, or navigate through endless charts and menus just to log a $5 coffee. I wanted a quick, distraction-free tool to jot down an expense, see the updated total instantly, and have it waiting for me the next day without any hassle.

<h2><a class="anchor" id="dataset"></a>Dataset</h2>

All data is generated by the user and saved securely right in their own browser using the Web Storage API (localStorage).

<h2><a class="anchor" id="tools--technologies"></a>Tools and Technologies</h2>

HTML5: To structure the app and form inputs.

CSS3: For the dark theme, flexbox layout, and hover effects.

JavaScript (ES6+): To handle the logic, DOM manipulation, and math calculations.

Web Storage API: Specifically localStorage to keep the data saved between sessions.

<h2><a class="anchor" id="methods"></a>Methods</h2>

Behind the scenes, the app takes the name and amount you type, validates them, and dynamically creates new list items on the page. I used a JavaScript array of objects to keep track of the expenses and synced it up with localStorage (using JSON.stringify and JSON.parse).

To calculate the total, I utilized the JavaScript array .reduce() method to sum up all the expense amounts efficiently. I also implemented event delegation on the expense list—instead of attaching a listener to every single delete button, I attached one to the parent <ul> and checked if the clicked target was a button.

<h2><a class="anchor" id="key-insights"></a>Key Insights</h2>

Using array methods like .reduce() makes calculating dynamic totals incredibly clean and straightforward compared to writing manual for loops.

Event delegation is a lifesaver for dynamically generated DOM elements. Handling the delete clicks at the parent level avoided the headache of constantly adding and removing event listeners.

localStorage is incredibly powerful for building quick, useful tools without needing to set up a whole backend database.

<h2><a class="anchor" id="dashboard"></a>Output</h2>

The final output is a clean, centralized interface. It has text and number boxes, an "Add Expense" button, your dynamic list of expenses, and a running total at the bottom. When you delete an expense, it is removed from the list and the total automatically recalculates.
